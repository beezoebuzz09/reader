# PDF Audiobook Reader

A local, offline-first PDF-to-speech reader that installs as an app on your iPhone home screen.

## Deploy to GitHub Pages

1. Create a new GitHub repo and push this folder to it (plain files, no build step needed):
   ```
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/<you>/<repo>.git
   git push -u origin main
   ```
2. On GitHub: **Settings → Pages → Source → Deploy from a branch**, pick `main` / `(root)`, save.
3. Wait a minute, then your app is live at `https://<you>.github.io/<repo>/`.

GitHub Pages serves over HTTPS, which is required for the service worker, Wake Lock, and Media Session APIs to work at all.

## Install on iPhone

1. Open the GitHub Pages URL in **Safari** (must be Safari, not Chrome — "Add to Home Screen" PWA install only works from Safari on iOS).
2. Tap the **Share** icon → **Add to Home Screen**.
3. Launch it from the home screen icon going forward — it opens full-screen, without Safari's UI chrome, like a native app.

## Background / lock-screen behavior

iOS Safari (even installed home-screen apps) suspends JavaScript speech synthesis when the screen fully locks or the app is backgrounded for a while — this is a platform restriction, not something a website can fully override. This app does everything that's possible client-side to work around it:

- **Screen Wake Lock** — while reading, the screen is kept from auto-locking/dimming, so the most common interruption (auto screen-off) doesn't happen.
- **Media Session API** — lock screen / Control Center / AirPods show play, pause, skip-back, skip-forward controls tied to the reader.
- **Silent keep-alive audio** — a real (silent) `<audio>` loop runs alongside speech, which gives iOS an active media session and makes background throttling less aggressive.
- **Auto-resume + "Tap to continue"** — your reading position is saved continuously. If iOS does kill speech while backgrounded, reopening the app tries to silently resume automatically; if iOS blocks that too, a one-tap "Continue reading" prompt appears so you pick up exactly where you left off instead of losing your place.
- **Resume across app restarts** — if you fully close and reopen the app later, it offers to resume the last PDF from the last sentence spoken (stored locally via IndexedDB — nothing leaves your phone).

If you manually press the physical lock button while reading, iOS may still pause speech until you unlock again — this is the one case no PWA can fully avoid, short of generating actual audio files server-side (a different architecture with its own tradeoffs: needs a TTS API, an API key, network access, and per-character cost). Either way, your position is saved continuously (via IndexedDB), so **⏯ Resume always continues from exactly where you left off** — even after the app is fully closed and reopened — while **▶ Play always restarts from the beginning**.

## Bookmarks

Tap **🔖 Bookmark** any time while reading to save your current spot. Tap **📑 Bookmarks** to see the list for the currently loaded book, jump back to any of them, or delete ones you don't need. Bookmarks are keyed to a fingerprint of the file's name + extracted text, so re-loading the same PDF later reunites you with the same bookmarks. Since this is a chunk-based reader (PDF text is flattened and re-split into speakable sentences), bookmarks point at a sentence position with a text snippet — not a PDF page number.

## Local development

Any static file server works (service workers require `http://localhost` or HTTPS, not `file://`):
```
python -m http.server 8000
```
Then open `http://localhost:8000`.

## Regenerating icons

Icons in `icons/` were generated by `scripts/make-icons.js` (pure Node, no dependencies). Re-run with `node scripts/make-icons.js` if you want to tweak the design.
